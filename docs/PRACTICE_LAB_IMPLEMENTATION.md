# Practice Lab Implementation

## Purpose

This document records the first real-content implementation of `M2.3 - Practice Lab`.

The Practice Lab preserves smaller coursework, experiments, and learning projects without allowing homework labels to define the main personal brand. HW numbering remains visible for traceability.

## Included projects

### HW6 - 50 Startups CRISP-DM Workflow

- Public GitHub Pages dashboard.
- Reproducible synthetic dataset.
- Linear regression pipeline.
- R2, MAE, RMSE, and 5-fold cross-validation.
- Feature-selection comparison.
- Technical whitepaper and visual gallery.
- Limitation: educational synthetic data, not real business research.

### HW5 - Machine Learning Top 10 Explorer

- Public GitHub Pages site.
- Ten machine-learning algorithm modules.
- Search and filters.
- Parameter sandboxes.
- Comparison matrix and quiz.
- Demo-only AI tutor interface.
- Limitation: the AI tutor does not call a real external API.

### HW3 - Cosmos Text-to-Image App

- Public Streamlit demo.
- Prompt, style, aspect-ratio, negative-prompt, seed, and image-count controls.
- Hugging Face token safety and Streamlit secrets support.
- Clearly labeled fallback demo mode.
- Limitation: the public model endpoint may be unavailable or too large for hosted inference.

### HW4 - Linear Regression Outlier Demo

- Local Streamlit demo.
- Adjustable synthetic-data parameters.
- Linear regression and residual analysis.
- Ranked top-ten outliers.
- CRISP-DM organization.
- Limitation: source and local demo only; no verified public deployment.

### HW1 - Real-time Clock Website

- Foundational static web exercise.
- HTML, CSS, and JavaScript date-time updates.
- Local verification and isolated GitHub publishing.
- Limitation: source record only in the current Practice Lab release.

## Public information rules

- Show a Live Demo button only when a verified public URL exists.
- Use a local-demo or source-record label when no public runtime exists.
- Keep limitations visible rather than presenting every exercise as production-ready.
- Keep original HW numbering for traceability.
- Promote a project to the main Portfolio only when its engineering story, evidence, and visual assets are strong enough.
